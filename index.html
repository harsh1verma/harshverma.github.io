Vulnerabilities

Selected web risks aligned with OWASP Top 10 2021 and advanced topics like SSRF and HTTP request smuggling, with practical mitigations and anchors for quick navigation.

Broken Access Control

Improper authorization checks enable IDOR, privilege escalation, or accessing restricted resources.

  • Mitigate with server-side authZ on every request; deny-by-default; robust object-level checks.
A01:2021OWASP

Cryptographic Failures

Weak or missing encryption, exposed secrets, and bad key management leak sensitive data.

  • TLS everywhere; vetted algorithms; key rotation; strong password hashing.
A02:2021OWASP

Injection

Untrusted input reaches interpreters (SQL/NoSQL/OS/LDAP), leading to data loss or RCE.

  • Parameterized queries, input validation, least-priv DB roles, safe APIs.
A03:2021OWASP

Insecure Design

Missing security requirements and threat modeling produce flawed controls.

  • Threat modeling, abuse cases, security NFRs and design reviews in SDLC.
A04:2021

Security Misconfiguration

Default creds, verbose errors, open buckets, or missing hardening baselines.

  • Secure baselines, hardened images, config scanning, minimize services.
A05:2021

Vulnerable & Outdated Components

Dependencies with known CVEs, unsupported versions, or untracked transitive risks.

  • SBOMs, version pinning, continuous scanning, patch cadence.
A06:2021

Identification & Authentication Failures

Weak auth flows, poor session handling, lack of MFA, credential stuffing risk.

  • MFA, secure sessions, rate limits, modern password storage.
A07:2021

Software & Data Integrity Failures

Untrusted CI/CD, unsigned updates, insecure deserialization break chain of trust.

  • Signed artifacts, supply chain verification, safe serialization formats.
A08:2021

Security Logging & Monitoring Failures

Insufficient logs and alerting delay detection and response.

  • Centralized logs, protected integrity, actionable alerts, retention.
A09:2021

Server-Side Request Forgery (SSRF)

Server fetches user-controlled URLs, reaching internal hosts or cloud metadata despite network ACLs.

  • Allowlist schemes/hosts/ports, block link-local/internal ranges, egress controls, validate/sanitize URLs.
A10:2021Cloud

HTTP Request Smuggling

Parser inconsistencies between proxies/backends (e.g., CL vs TE) enable desync, cache poisoning, and control bypass.

  • Standardize parsing, avoid ambiguous headers, align front/back ends, prefer HTTP/2 backhaul where feasible.
DesyncProxy Chains

Cross-Site Scripting (XSS)

Reflected/stored/DOM payloads execute in user context.

  • Output encoding, CSP with nonces, input validation, safe templating.

Cross-Site Request Forgery (CSRF)

Forged state-changing requests using victim’s session.

  • SameSite cookies, anti-CSRF tokens, re-auth for sensitive actions.

SQL Injection (SQLi)

Manipulated queries exfiltrate/alter data and escalate privileges.

  • Parameterized queries, least-priv DB users, input validation.

Security Headers

Missing CSP, HSTS, or frame protections expose users to client-side risks.

  • Set CSP, HSTS, X-Frame-Options, Referrer-Policy; validate via scanners.

Insecure Deserialization

Unsafe object deserialization leads to code execution or logic abuse.

  • Avoid native deserialization for untrusted data; use safe formats and integrity checks.
